# Инструкции по восстановлению проекта

## Способ 1: Восстановление через Git тег (рекомендуется)

Если у вас есть доступ к git-репозиторию, восстановление очень простое:

```bash
# Перейти в директорию проекта
cd B:\telegram_planfix_bot\telegram_planfix_bot

# Посмотреть все доступные теги
git tag -l

# Вернуться к версии с тегом v1.0-stable-2025-11-17
git checkout v1.0-stable-2025-11-17

# Или создать новую ветку от этого тега
git checkout -b restore-stable-2025-11-17 v1.0-stable-2025-11-17
```

## Способ 2: Восстановление из архива

1. Распаковать архив `telegram_planfix_bot_backup_2025-11-17.zip` в нужную директорию
2. Установить зависимости: `pip install -r requirements.txt`
3. Настроить переменные окружения (файл `.env`)
4. Запустить проект

## Отправка тега в удаленный репозиторий

Чтобы тег был доступен на удаленном сервере:

```bash
# Отправить конкретный тег
git push origin v1.0-stable-2025-11-17

# Или отправить все теги
git push origin --tags
```

## Важные файлы для восстановления

- `requirements.txt` - зависимости Python
- `.env` или `.env.example` - переменные окружения (ключи API, токены)
- `database.db` или скрипты миграции БД - база данных
- `config/` - конфигурационные файлы
- `*.py` - исходный код
